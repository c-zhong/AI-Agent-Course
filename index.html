Introduction to AI Agents

A hands-on introduction to autonomous AI systems — concepts, architecture, tools, and real-world applications for business analytics.

What Are AI Agents?

An AI agent is a software system that perceives its environment, plans actions, uses tools, and autonomously works toward a goal — often completing multi-step tasks without constant human input.

💡
One-line definition: An AI agent = a Large Language Model (LLM) + memory + tools + the ability to plan and act on its own, step by step.
The evolution: from chatbots to agents
Feature Simple Chatbot LLM (e.g., ChatGPT) AI Agent
Remembers conversation history ✗ No ✓ Yes ✓ Yes + long-term
Can use external tools (search, code, APIs) ✗ No ~ Rarely ✓ Core feature
Plans multi-step actions autonomously ✗ No ✗ No ✓ Yes
Can correct itself when something fails ✗ No ✗ No ✓ Yes
Works without step-by-step human prompting ✗ No ✗ No ✓ Yes
Can coordinate with other AI agents ✗ No ✗ No ✓ Multi-agent possible
A concrete analogy

Think of the LLM as a brilliant but forgetful consultant who can answer any question. An AI agent gives that consultant a desk, a computer, sticky notes, access to databases, a phone, and an assignment — and says: "Go solve this problem. Come back when you're done."

Without an agent (LLM only)
You: "Analyze our Q3 sales drop."
LLM: Here's some general advice about sales... ✗
Can't access your data, can't run numbers
With an agent
You: "Analyze our Q3 sales drop."
Agent: ① Fetches your CRM data → ② Runs analysis → ③ Compares to Q2 → ④ Returns insight ✓
Key characteristics of AI agents
🎯 Goal-directed
Works toward a high-level objective, not just one response at a time
🔧 Tool-using
Can call APIs, run code, search the web, read files
🗂️ Memory
Remembers context, past actions, and accumulated knowledge
🔄 Iterative
Perceives results, reflects, and adjusts its next action

Agent Architecture

Every AI agent — no matter the framework — is built from the same five core components. Understanding these is the foundation for designing your own agent.

🧠 LLM Brain

The reasoning engine. It reads instructions, decides what to do next, and generates language. This is Claude, GPT-4, Gemini, etc.

🗂️ Memory

Short-term (current conversation) + long-term (vector databases that store facts, past interactions, and domain knowledge).

🔧 Tools

Functions the agent can call: web search, code execution, database queries, email sending, API calls. This is what gives agents real-world power.

🎯 Planning

Breaking a big goal into steps. Techniques: ReAct (Reason + Act), Chain-of-Thought, Tree-of-Thought. The agent decides what to do next.

📥 Perception / Input

Text, images, file contents, database records, tool results — anything the agent can read and process as context.

📤 Action / Output

What the agent produces: a report, a decision, an API call, code to run, or a message sent — the agent's effect on the world.

🔄
The Agent Loop: At its core, an agent runs a simple cycle — Observe → Think → Act → Observe → ... — until the task is complete or the agent decides it needs more information from you.

# Step 3: Write the actual Python functions that do the real work
# In a real project, these would call your database or APIs

def get_company_data(company_name: str) -> dict:
    # In production: query your CRM, ERP, or data warehouse
    database = {
        "TechCorp": {"revenue": 5_200_000, "profit_margin": 0.22, "growth": 0.18},
        "RetailCo": {"revenue": 12_000_000, "profit_margin": 0.08, "growth": 0.04},
        "HealthPlus": {"revenue": 3_800_000, "profit_margin": 0.31, "growth": 0.25},
    }
    return database.get(company_name, {"error": "Company not found"})

def calculate_roi(investment: float, annual_return: float, years: int) -> dict:
    total_return = annual_return * years
    roi_pct = ((total_return - investment) / investment) * 100
    return {
        "total_return": total_return,
        "net_profit": total_return - investment,
        "roi_percent": round(roi_pct, 2),
        "payback_years": round(investment / annual_return, 1)
    }

def get_industry_benchmark(industry: str, metric: str) -> dict:
    benchmarks = {
        "tech": {"profit_margin": 0.18, "revenue_growth": 0.15},
        "retail": {"profit_margin": 0.06, "revenue_growth": 0.05},
        "healthcare": {"profit_margin": 0.12, "revenue_growth": 0.10},
    }
    result = benchmarks.get(industry.lower(), {})
    return {"industry": industry, "metric": metric, "benchmark": result.get(metric, "N/A")}

# Route tool calls to the right Python function
def execute_tool(tool_name: str, tool_input: dict) -> str:
    if tool_name == "get_company_data":
        result = get_company_data(**tool_input)
    elif tool_name == "calculate_roi":
        result = calculate_roi(**tool_input)
    elif tool_name == "get_industry_benchmark":
        result = get_industry_benchmark(**tool_input)
    else:
        result = {"error": f"Unknown tool: {tool_name}"}
    return json.dumps(result)
PYTHON — THE AGENT LOOP ⭐
# Step 4: The agent loop — this is the heart of every AI agent
# The agent keeps going until it has a final answer

def run_agent(user_question: str) -> str:
    messages = [{"role": "user", "content": user_question}]
    
    while True:
        # Call Claude with the available tools
        response = client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=1024,
            tools=tools,
            messages=messages,
            system="You are a Business Intelligence Agent. Use tools to gather data, then synthesize a clear insight."
        )
        
        # If Claude is done reasoning, return the final answer
        if response.stop_reason == "end_turn":
            return response.content[0].text
        
        # Otherwise, Claude wants to call a tool — execute it and loop
        tool_results = []
        for block in response.content:
            if block.type == "tool_use":
                result = execute_tool(block.name, block.input)
                print(f"Tool called: {block.name} → {result}")
                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": result
                })
        
        # Add the tool results to the conversation and keep looping
        messages.append({"role": "assistant", "content": response.content})
        messages.append({"role": "user", "content": tool_results})

# Run it!
answer = run_agent("Analyze TechCorp profitability vs the tech industry average")
print(answer)
📝
Key insight — the agent loop: Notice the while True loop. The agent keeps running until stop_reason == "end_turn". Every iteration: Claude decides what tool to call → you execute it → you give the result back → Claude decides the next step. This loop is the agent.
